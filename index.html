I will keep these four goals

  • 웹 접근성 준수

    웹 접근성을 준수하여 모든 사용자가 콘텐츠에 쉽게 접근할 수 있도록 구현하겠습니다.

  • 사용자 중심의 웹 경험 구현

    웹사이트를 제작할 때 사용자 경험을 최우선으로 고려하며, 직관적인 디자인을 구현하는것에 집중하겠습니다.

  • 일관된 사용자 경험 제공

    반응형 웹 디자인을 활용해 다양한 환경에서 최적화된 디자인을 구현하여 기기 간의 사용자 경험 차이를 최소화하겠습니다.

  • 최신 웹 트렌드에 발전을 추구

    최신 웹 트렌드에 발맞추어, CSS Grid와 JavaScript ES6+, 그리고 React를 활용해 효율적인 클린코드를 작성하고 있습니다.

웹 접근성과 문서 구조를 고려해 모든 사용자가 쉽게 콘텐츠에 접근할 수 있도록 하고, 정확한 HTML과 시맨틱 마크업으로 정보 전달과 SEO를 동시에 잡겠습니다. 하지만 단순한 구현을 넘어, 직관적인 디자인과 편리한 탐색이 가능한 사용자 경험을 최우선으로 하며, 반응형 웹을 통해 어떤 기기에서도 일관된 경험을 제공하는 것에 집중합니다. 변화하는 트렌드 속에서 JavaScript와 React를 활용해 효율적인 클린 코드를 작성하고, 개발자와 기획자와의 원활한 소통을 통해 최상의 결과물을 만들어내는 것이 목표입니다.

배열

#Javascript #Array

들어가기에 앞서... 배열 : 여러개의 값을 순차적으로 나열한 자료구조 Javascript 에서의 배열과 일반 컴퓨터 언어에서의 배열은 다르다. 일반 자료구조에서 말하는 일반적인 배열 동일한 크기의 메모리 공간이 빈틈없이 연속적으로 밀집하여 나열된 자료구조 하나의 데이터 타입으로 통일되어있다. 장점 산술식 연산으로 한번에 요소에 빠르게 접근할수 있다.( 효율적으로 요소에 접근할수 있다) 인덱스가 0 인, 요소의 메모리 주소 : 1000 + 0 (인덱스) * 8(요소의 바이트) = 1000 인덱스가 1인, 요소의 메모리 주소 : 1000 + 1 (인덱스) * 8(요소의 바이트) = 1008 인덱스가 2인, 요소의 메모리 주소 : 1000 + 2 (인덱스) * 8(요소의 바이트) = 1016
